# Internationalization path (v2)

v1 ships **English-first**: system prompt, taxonomy labels, UI chrome, and heuristic patterns.

## Planned approach

1. Extract UI strings to `locales/en/messages.json` (Chrome i18n) for popup / options / side panel.
2. Keep bias **type enums** stable in English (`loaded_language`, …) for schema compatibility; localize **labels** only.
3. System prompt: optional `response_language` field so explanations match the article language while enums stay fixed.
4. Heuristics: per-locale pattern packs (start with es, de, fr).
5. RTL: ensure tooltip/side panel layout uses logical CSS (`inset-inline`, etc.).

## Non-goals for v1

- Auto-detecting article language for model routing  
- Translating primary-source citations  

Contributions for locale packs are welcome once the `chrome.i18n` scaffold lands.
